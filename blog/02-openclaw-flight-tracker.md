# OpenClaw로 항공권 가격 트래커 만들기 (Mac mini 24/7 운영)

> 1편에서 만든 Mac mini + OpenClaw 환경 위에, 실제 자동화 서비스를 올린 구현기

## 목표

매번 항공권 사이트를 직접 확인하는 대신,

- 정해진 노선의 가격을 주기적으로 수집하고
- 최저가 변동이 있으면 즉시 알림 받고
- 하루 4번 요약 브리핑을 받는 것

을 목표로 만들었다.

---

## 아키텍처

구성은 단순하게 유지했다.

1. **Playwright 크롤러** (`tracker.py`)
2. **SQLite 저장소** (`flight_tracker.db`)
3. **Discord 알림** (즉시/정기)
4. **Vercel 대시보드** (data.json 기반)

흐름:

- 크롤러 실행 → DB 업데이트 → `data.json` export → GitHub push → Vercel 반영

## 추적 조건

### 정기 구간 (ROUTES)
- ICN → FUK
- ICN → NRT
- GMP → HND

### 특별 구간 (SPECIAL_ROUTES)
- ICN → DPS (발리)
- ICN → PQC (푸꾸옥)
- ICN → HKT (푸켓)

### 필터
- 직항만
- 동일 항공사 왕복만
- (정기 구간) 출발 18시 이후 / 귀국 16시 이후
- 스캔 범위 16주

### 날짜
- 기본 패턴: 금→일
- `SPECIAL_DATES`로 연휴 1회성 날짜 추가
- `SPECIAL_ROUTES`는 별도 날짜 세트 지정

## 구현 포인트 1) 파서 안정화

초기엔 "결과 없음"이 자주 나왔다. 개선한 포인트:

- 출발 시간 패턴 기반 블록 탐지 (`HH:MM{ORIGIN}`)
- `직항` 판별 범위를 `i+2..i+4`까지 확장 (+1일 케이스 대응)
- 메타 텍스트 키워드 제외
- 혼합 항공사 왕복 조합 제거

스크래핑은 결국 "한 번 되는 코드"보다 "운영 중에도 버티는 파서"가 중요했다.

---

## 구현 포인트 2) 3인 가격(pax3_price)

실제 구매 시 체감가를 보려고 3인 가격도 저장했다.

- `NULL`: 조회 실패/미확인
- `-1`: 동일 항공사 3석 불가
- `양수`: 3인 예약 시 1인당 가격

덕분에 최저가 카드에서
"1인 기준 최저"와 "3인 동시 예매 가능 가격"을 같이 비교할 수 있다.

---

## 구현 포인트 3) 데이터 모델

핵심 테이블:

- `weekly_lowest`: 날짜 조합별 현재 최저가
- `scan_history`: 스캔 로그
- `price_history`, `weekly_price_history`: 시계열

정리 정책:

- 과거 출발일 `weekly_lowest`는 정리
- 오래된 `scan_history`는 보관 기간 기준 정리
- 삭제 권한은 `tracker.py`로 단일화 (브리핑 스크립트는 삭제 금지)

---

## 구현 포인트 4) OpenClaw 크론 운영

실운영 스케줄:

- 매시 정각: 크롤러
- 09/13/17/21시: 브리핑

경험상 핵심은 3가지:

1. **타임아웃 관리**: 브리핑은 hang 감지 필요
2. **권한 관리**: Discord 채널 private 전환 시 봇 접근 재설정
3. **장비 정책**: Mac mini 수면/절전 영향 점검

---

## 구현 포인트 5) 대시보드

- Next.js + Vercel
- 크롤러가 `data.json`을 push
- 대시보드가 이를 렌더링

주의사항 하나:

> 대시보드 코드 변경은 GitHub push만으로 자동 반영되지 않아서, `vercel --prod` 수동 배포가 필요했다.

---

## 운영하면서 배운 점

1. 자동화는 "코드"보다 "운영"이 오래 걸린다.
2. 장애 포인트는 로직보다 권한/타임아웃/절전에서 자주 터진다.
3. OpenClaw를 쓰면 "개발"과 "정기 실행"을 한 컨텍스트에서 다루기 쉬워진다.

---

## 왜 OpenClaw였나 (한 줄 요약)

**Mac mini 위에서 돌아가는 에이전트를 중심으로, 크롤링·스케줄·알림·운영을 한 번에 묶기 좋아서**.

이번 프로젝트는 "AI가 코드를 써줬다"가 아니라,

> OpenClaw를 운영 레이어로 두고 자동화 시스템을 실제로 굴린 사례

에 더 가깝다.
